Guardian.js

A lightweight, zero-dependency browser security package with debugger traps, devtools detection, anti-inspect, and more.

Guardian is not initialized. INACTIVE
🪤

Debugger Trap

Fires a debugger statement on a rapid interval using the Function constructor, pausing execution whenever DevTools is open and making the page unusable to inspectors.

Aggressive setInterval Function constructor
🔍

DevTools Detection

Uses four independent heuristics — window size delta, debugger timing, toString override, and Firebug legacy check — to reliably detect when DevTools is open.

Size delta Timing toString trick Polling
🚫

Anti-Inspect

Blocks F12, Ctrl+Shift+I, Ctrl+Shift+J, Ctrl+Shift+C, Ctrl+U, Ctrl+S, and right-click context menu — every common pathway to browser developer tools.

F12 blocked Right-click off View-source blocked
🖥️

Console Shield

Overrides all console methods (log, warn, error, table, dir, etc.) with no-ops so your application does not leak sensitive information through the browser console.

console.log console.warn console.error Restorable
📋

Anti-Select & Anti-Copy

Prevents text selection via CSS and event listeners, and blocks clipboard copy/cut operations so page content cannot be easily extracted.

userSelect: none selectstart copy/cut events
⚡

Action Engine

When DevTools is detected, choose your response: blur the page, redirect to another URL, clear all content, show a warning, or trigger a fully custom callback.

blur redirect clear custom
🤖

Headless Detection

Detects Puppeteer, Playwright, PhantomJS, and Selenium by checking navigator.webdriver, plugin count, user-agent strings, and missing browser APIs common in automation environments.

navigator.webdriver UA check plugin count
🖼️

Anti-Iframe

Detects when the page is loaded inside a foreign iframe and immediately busts out. If the framebusting is blocked by a sandbox attribute, the document body is wiped instead.

Frame bust top !== self Sandbox fallback
🔒

Tamper Detection

Snapshots the string representation of critical Guardian functions on init, then re-checks them every second. Fires the action engine if any function has been patched or replaced at runtime.

Function snapshot Polling toString check
💧

DOM Watermark

Injects a unique per-session token into a hidden DOM node and a <meta> tag on every page load. Allows scraped HTML to be traced back to a specific session.

Session token meta tag Hidden node

Usage

Basic

Modules
// Include the script
<script src="guardian.js"></script>

// Initialize with defaults (recommended)
Guardian.init();

// Or pick specific features
Guardian.init({
  debuggerTrap:      true,
  antiInspect:       true,
  devToolsDetection: true,
  consoleShield:     false,
  antiSelect:        false,
  antiCopy:          false,
  action: 'warn',
});
// Advanced: custom callback + redirect
Guardian.init({
  debuggerTrap:      true,
  devToolsDetection: true,
  antiInspect:       true,
  consoleShield:     true,
  antiCopy:          true,
  action:            'custom',
  trapInterval:      50,
  detectionInterval: 500,

  onDevToolsOpen: function() {
    // fired once when devtools opens
    document.getElementById('warning').style.display = 'flex';
  },

  onDevToolsClose: function() {
    // fired once when devtools closes
    document.getElementById('warning').style.display = 'none';
  },
});
// Use individual modules directly
const { DebuggerTrap, DevToolsDetector, AntiInspect,
         ConsoleShield, ContentProtection } = Guardian.modules;

// Start debugger trap at 50ms
DebuggerTrap.start(50);

// Poll for devtools every 500ms
DevToolsDetector.startPolling(500, onOpen, onClose);

// Block right-click & shortcuts
AntiInspect.enable();

// Silence all console output
ConsoleShield.enable();

// Prevent copy / select
ContentProtection.enableAntiCopy();
ContentProtection.enableAntiSelect();

// Teardown everything
Guardian.destroy();
